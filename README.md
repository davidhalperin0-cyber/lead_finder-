# 🔍 מנוע חיפוש לידים - אתרים מיושנים

כלי שמוצא עסקים מקומיים עם אתרים ישנים, מנתח אותם, ומחזיר לך טבלת אקסל מסודרת עם **ציון התיישנות, פרטי קשר, ותבניות פנייה מוכנות**.

המודל העסקי: לוקחים את הרשימה, פונים בוואטסאפ/טלפון, ובונים להם אתר חדש בעזרת AI במחיר זול.

## 📊 מה מקבלים? (ראי `sample_output.xlsx`)

קובץ אקסל עם 4 גליונות:

### גליון 1: לידים
טבלה ממוינת מהאתרים הכי "ישנים" ועד הכי טובים. כל שורה כוללת:
- **ציון** (0-100) ו**אות** (F=ישן מאוד, A=מעולה)
- **שם עסק, אתר, אימייל, טלפון, וואטסאפ, פייסבוק, אינסטגרם, כתובת**
- **בעיות עיקריות** של האתר (5 הראשונות)
- האם יש HTTPS, האם מותאם למובייל, איזה CMS

### גליון 2: סיכום
ספירה אוטומטית של כמה לידים בכל ציון.

### גליון 3: תבניות פנייה
**5 תבניות מוכנות לשימוש בעברית:**
1. הודעת WhatsApp ראשונית קצרה
2. מייל ראשון (פנייה קרה)
3. פתיח לשיחת טלפון (15 שניות)
4. תגובה ל"אין לי תקציב"
5. הצעת מחיר מסודרת

### גליון 4: מתודולוגיה
מסביר בדיוק איך מחושב הציון - כמה נקודות יש על "אין מובייל", כמה על "Flash" וכו'. שימושי כשמסבירים ללקוח למה האתר שלו צריך שדרוג.

## 💡 איך להפיק ממנו הכי הרבה כסף

### 1. תתחילי עם **F ו-D** (ציון 60+)
אלה הלידים החמים ביותר. האתר שלהם באמת לא עובד נכון, וקל לשכנע אותם.

### 2. **פני בוואטסאפ קודם, לא במייל**
שיעורי תגובה גבוהים פי 5. תשתמשי בתבנית מהגליון "תבניות פנייה".

### 3. **תכיני "תצוגה מקדימה"** לפני השיחה
לפני שמתקשרים ללקוח, תפעילי AI (Claude/GPT/v0) ותכיני מוקאפ של איך האתר *יכול* להיראות. זה מוכר את עצמו.

### 4. **מחיר נכנס: 1,500-2,500 ₪ לאתר בסיסי**
- חצי מקדמה, חצי בסיום
- 5-7 ימי עסקים
- אחסון ודומיין שנה ראשונה כלולים
- אופציה לחבילת תחזוקה חודשית של 200-400 ₪

### 5. **תזכרי שזה משחק מספרים**
מתוך 100 פניות, נורמלי לסגור 3-5 עסקאות. אם סוגרות 5 בחודש = הכנסה של 7,500-12,500 ₪ בחודש.

### 6. **תבני פייפליין**
- שבוע 1: מריצה את הסקריפט, אוספת 100-200 לידים
- שבוע 2: פונה ל-50 הראשונים בוואטסאפ
- שבוע 3: שיחות פולואפ + הצעות מחיר
- שבוע 4: סוגרת את אלה שמתבשלים, מתחילה בנייה

## ⚠️ הערות חשובות

### זמן ריצה
~60 לידים לוקח כ-2-4 דקות (תלוי במהירות האתרים). הסקריפט סורק 5 אתרים במקביל.

### מקורות חיפוש
1. **OpenStreetMap** (ראשי) - חינמי, עובד טוב, מכסה הרבה עסקים בישראל
2. **DuckDuckGo** (גיבוי) - אם OSM לא מצא, חיפוש כללי

### אם לא נמצאו עסקים
- נסי שם עיר באנגלית ובעברית (`"Tel Aviv"` במקום `"תל אביב"`)
- נסי סוג עסק יותר רחב (`"shop"` במקום `"חנות נעליים"`)
- נסי ערים גדולות יותר (תל אביב, חיפה, ירושלים מכוסות הכי טוב)

### חוקיות
הסקריפט מוריד דפים פומביים בלבד (כמו שגוגל עושה). אין כאן שום פעולה לא חוקית.
**עבור שליחת מיילים שיווקיים בישראל - חוק הספאם דורש "אופט-אין"**, אז עדיף להתחיל בשיחה/וואטסאפ מאשר במיילים מאסיביים.

## Web App (Next.js + FastAPI + Supabase)

מערכת מלאה בדפדפן (מחשב + מובייל) — בלי להריץ סקריפטים ידנית.

### 1) Supabase
1. צרי פרויקט ב-[Supabase](https://supabase.com).
2. **Authentication** → Email — הפעילי כניסה עם מייל+סיסמה; צרי משתמש אחד (רק את).
3. הריצי את הקובץ `supabase/migrations/20260228120000_init_lead_crm.sql` ב־SQL Editor.
4. מהגדרות הפרויקט העתיקי: `URL`, `anon key`, `service_role`, ו־**JWT Secret**.

### 2) שרת API (Python)
```bash
cd api
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
# העתיקי api/.env.example ל-api/.env ומלאי מפתחות + OPENAI_API_KEY
cd .. && pip install -r requirements.txt   # לוגיקת find_leads בשורש
cd api
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 3) פרונט (Next.js)
```bash
cd web
cp .env.local.example .env.local
# מלאי NEXT_PUBLIC_* וכתובי את כתובת ה-API
npm install
npm run dev
```
הערה: מומלץ **Node 20 LTS**. אם אצלך Node 24, ייתכן ש-`next dev` ייתקע. בתיקייה `web/` יש `.nvmrc` שמגדיר 20.
פתחי [http://localhost:3000](http://localhost:3000) — התחברי עם המייל מ-Supabase Auth.

### זרימה
- **חיפוש חדש** → קורא ל־`POST /api/search-leads` (מריץ את `run_pipeline` ושומר לידים ב-Supabase).
- **רשימת לידים / פירוט** → קריאות ישירות ל-Supabase (RLS) + עדכון סטטוס דרך `PATCH /api/leads/:id`.
- **ייצוא** → `GET /api/export/csv` עם אותם מסננים (פתחי ב-Excel).

**אין** שליחת הודעות אוטומטית — רק כפתורי `tel:` / וואטסאפ ידניים והעתקת טקסט.

## Stability checklist (יומיומי)

- **login עובד**: אפשר להתחבר עם המשתמש היחיד ב-Supabase Auth.
- **health API**: פתיחת `GET /api/health` מחזירה `ok: true` ושה-ENV קיים.
- **search עובד**: מסך “חיפוש חדש” → job הופך ל-`running` ואז `completed`.
- **job progress ברור**: רואים `found / analyzed / saved / errors`.
- **leads נשמרים**: Dashboard מציג כרטיסים (כולל `opening_line` וסטטוס).
- **lead detail עובד**: כרטיס קרב בולט, `tel:` עובד באייפון, WhatsApp נפתח ידנית, copy עובד.
- **status update עובד**: “התקשרתי/חם” מהדשבורד + Save בעמוד ליד.
- **export עובד**: CSV לפי מסננים כולל עמודות: name, website, phone, whatsapp, score, grade, priority, strongest_problem, opening_line, status, notes, follow_up_date.

## Data quality notes
- המערכת עושה **dedupe** לפי `(user_id, site_key)` (host+path נקי) כדי למנוע כפילויות.
- טלפונים מנוקים ל-`+972...` ו-URLs מנוקים לפני שמירה.
- לידים ללא אתר לא נשמרים.
